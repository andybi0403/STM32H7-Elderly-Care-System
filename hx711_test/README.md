# HX711 三路称重传感器 + LCD 显示

基于 STM32H7R3 裸机，GPIO 模拟 HX711 协议，TFT LCD 实时显示三路重量。

## 接口定义

### 传感器 → 开发板

| | 传感器1 | 传感器2 | 传感器3 |
|------|---------|---------|---------|
| **DOUT** | **PB6** | **PB8** | **PA4** |
| **SCK** | **PB7** | **PB9** | **PA7** |
| **VCC** | DAPLink 3.3V | DAPLink 3.3V | DAPLink 3.3V |
| **GND** | DAPLink GND | DAPLink GND | DAPLink GND |

> 所有引脚均在 P2 排针上。三个 HX711 共享 DAPLink 3.3V 供电，共地。

### 称重传感器 → HX711 绿端子

| 传感器线 | HX711 | 说明 |
|---------|-------|------|
| 红线 (激励+) | **E+** | 桥式传感器供电 |
| 黑线 (激励-) | **E-** | |
| 白线 (信号-) | **A-** | 差分信号 |
| 绿线 (信号+) | **A+** | |

> 颜色以实物标注为准。若读数反方向变化，对调 A+/A-。

### 屏幕显示

药品页面显示三路重量：

| 字段 | 传感器 |
|------|--------|
| **W1** | 传感器1 (PB6/PB7) |
| **W2** | 传感器2 (PB8/PB9) |
| **W3** | 传感器3 (PA4/PA7) |

## 行为

1. 上电后等待 1 秒，丢弃 10 个不稳定读数
2. 取 10 次平均作为零点
3. 三路传感器空载时均接近 0
4. 重量变化时自动刷新屏幕，无变化不刷新（无闪烁）
5. 每秒约读取 5 次

## 编译

```bash
cd firmware/module_tests/hx711_test
make -f Makefile_hx711 clean all
```

## 加载运行 (SRAM)

```bash
python load_to_sram.py
```

> Flash 烧录暂不支持 (H7RS 安全限制)，断电后需重新加载。

## 故障排查

| 现象 | 可能原因 |
|------|---------|
| 屏幕不亮 | DAPLink 重插，重新加载 |
| 某路读数一直为 0 | HX711 VCC/GND 未接 |
| 某路读数非常大且不变 | DOUT/SCK 未接或接反 |
| 某路读数不断递增 | HX711 模块故障，更换 |
| 读数方向反了 | 对调 A+/A- |

## H7RS 注意事项

- RCC_AHB4ENR 在 **+0x140** (非标准 H7 的 +0x50)
- SRAM 运行需手动清零 BSS
- 调试器不能直接写 GPIO，须通过 CPU 代码操作
- GPIOA=0x58020000, GPIOB=0x58020400
